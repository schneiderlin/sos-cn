# Fact Check 指南

## 原则
**代码永远是真相基准**。当文档与代码冲突时，代码正确。

## 流程
1. **多源验证**：对比文档、Clojure 包装层、底层 Java 源码
2. **参数检查**：验证实际接受的参数 vs 文档声称的参数
3. **模式对比**：参考类似模块的实现模式
4. **修复方向**：修正文档，而非代码

## 方法
- 读取实现文件（如 `well.clj`）
- 检查底层引擎（如 `WellInstance.java`）
- 对比类似模块（如 `hearth.clj`）
- 确认参数真实存在且可配置

## 结果
记录不匹配项，提供修复建议。
